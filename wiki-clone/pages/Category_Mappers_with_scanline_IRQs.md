# Category:Mappers with scanline IRQs

From [NESdev Wiki](Nesdev_Wiki.xhtml) | View [current version](https://www.nesdev.org/wiki/Category%3AMappers_with_scanline_IRQs) | View [other pages](Special_AllPages.xhtml#Category_Mappers_with_scanline_IRQs)

Mappers in this category have the ability to schedule an IRQ an arbitrary and continuously-adjustable number of scanlines in the future. This is triggered in response to a rendering signal such as [PPU A12](PPU_pinout.xhtml "PPU pin out and signal description"). 

Mappers with configurable [cycle-based IRQs](Category_Mappers_with_cycle_IRQs.xhtml "Category:Mappers with cycle IRQs") can simulate a scanline-counter by waiting an equivalent number of CPU cycles, but have less robust horizontal timing, since they will fire relative to their start time, rather than relative to a rendering signal. The [VRC IRQ](VRC_IRQ.xhtml "VRC IRQ") has a convenient mode for this. 

Mappers with a fixed delay (but still clocked by the PPU) should go in the [fixed-timing category](Category_Mappers_with_fixed_timing_scanline_IRQs.xhtml "Category:Mappers with fixed-timing scanline IRQs") instead.
